Mit welchen fachlichen Argumenten wurde das WWW-Proposal von Tim Burners Lee abgelehnt?

  • Sein Chef hat die Vagheit bzw. die Idee nicht verstanden und mit Eolvierbarkeit verwechselt

  • HyperText-Community war in der Wissenschafft weiter fortgeschritten, als das Konzept von Tim berners-lee anfangs anbat und ein rückschritt wäre

  • Broken links: da keine referenzielle Integrität wegen der Dezentralen Verwaltung von Web Sites und Ressourcen auf verschiedenen Webservern, kann eine URL zwar syntaktisch korrekt, jedoch nicht funktionieren


Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?

  • Ortogonale Technologien: www lässt sich in drei verschiedenen Richtungen( HTML, HTTP/HTTPS und URI) weiterentwickeln. entwicklung ohne zentrale Kontrolle

  • Ermöglicht dezetrale Verwaltnung von Ressourcen und Webseiten.

Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?

  • Anfrage war erfolgreich und Server hat die Daten erfolgreich geliefert

Anfrage war erfolgreich und Server hat die Daten erfolgreich geliefert

  • Angeforderte Ressource wurde verlegt. Daher sollte der Browser automatisch weiterleiten, falls im HTTP-Header der Server-Antwort die neue URL steht für die (dauerhafte) Weiterleitung hinterlegt wurde.

Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?

  • "Bad Request" => Anfrage war fehlerhaft oder nicht verstanden wurde. Daher sollte man sleber überprüfen ob die Abfrage korrekt ist und ggf. Fehler beheben


Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?

  • "Forbidden" => Server hat Anfrage zwar verstanden, kann aber aus irgendeinem Grund nicht auf die angeforderten Ressourcen zugreifen bzw. wird ihm verweigert.
    Kann an Berechtigungsproblem liegen. Daher sichersrtellen, dass man die Berechtigung des Zugriffes hat oder die IP-Adresse nicht beschränkt ist durch durch bspw. GeoBlocking

Was ist HTML?

  • HTML(Hyper Text Mark-Up Language ) ist eine textbasierte, kontextfreie (Chonsky Typ 2, Kellerautomat) Auszeichnungssprache, für Strukturierung von digitalen Dokumenten.


Wie kann man eine geschachtelte geordnete Liste der Tiefe 3 erzeugen?

-Vorab: "(ul)...(/ul)" oder "(ol)...(/ol)" sind Block-Tags und erstellen eine Liste. Ersteres eine unsortierte (Stichpunkte), zweiteres eine sortierte (nummerierte Aufzählung) Liste.

Bei beiden muss innerhalb für jedes Element der Liste "(li) x (/li)" stehen, wobei x für den Inhalt bzw. dem "String" oder anderen HTML-kompatible Medium steht, der angezeigt werden soll.
Um nun eine sortierte, verschachtelte Liste mit tiefe n zu erzeugen, muss man innerhalb der aktuellen Verschachtelungstiefe, einfach eine weitere sotirete Liste erzeugen.
Für die äußere Liste sind innerhalb ihrer sowohl "normale" Elemente, als auch Listen gleich gesetzt bzw. -gültig.

Wie ist eine HTML-Tabelle aufgebaut?

Bsp.

  1. Zeile:

    • Table Row ( kurz tr), um eine Zeile zu definieren:"(tr) ... (/tr)"

  2. Namen der Spalten / Table Header:

    • mit "(th) ... (/th)" kann man den Namen der Spalte festlegen
    • steht innerhalb eines "(tr) ... (/tr)"

  3. Spalte:

    • Table Data (kurz td), um innerhalb einer Zeile die daten abzugeben
    • nutzt dafür "(td) ... (/td)" , um die Spalteninhalte innerhalb einer Zeile Anzugehen
    • daraus ergeben sich automatisch die Spalten

    • Tabellen sind standesgemäß quadratisch. Falls also Zeilen unterschiedlich groß sind, wird die Spaltenanzahl der Table bzw. Breite der Tabelle anhand der Zeile, die am meisten Spalten benötigt, bestimmt.
      => Leere Einträge bei den anderen Zeilen, wo keine Einträge für die Zeile stehen

    • steht innerhalb eines "(tr) ... (/tr)"

Vorgehen

Für jede Verschachtelte Tabellen, in dem man innerhalb eines "(td) ... (/td)" einfach so vorgehen, als würde man eine neue Tabelle erstellen

Hyperlinks

nutzt dafür "(a href=...)"

  • der "Anocher-Tag" "(a ...)"" wird verwendet, um Hyperlinks zu erstellen, um auf andere digitale Dokumente bzw. Webseiten zu gelangen (falls ja das digitale Dokument nicht auf der aktuellen Webseite sich befindet)

    das href-Attribut ( Hypertext Reference) dient dazu, das Ziel des Hyperlinks anzugeben Bsp.(a href="https://www.example.com")Besuche

  • falls bspw. "#news" steht wie bspw. in "(a href="#news")News(/a)", handelt es sich um ein internen Verweis, daher der link zu einem anderen abschnitt der Webseite bzw. des digitalen Dokuments verweist

    dafür muss aber zuvor ein hyperlin innerhalb des domukentes erstellet worden sein und mus wie folgt aussehen:

    (h1)(a name="section1")(/a)Erster Abschnitt(/h1)

    (a href="#section1")Zu Abschnitt 1 springen(/a)
    nun wäre solch ein interner Verweis möglich


Welche Attribute sollte man bei Bildern wie verwenden?

Vorab: mit"(img src=... alt=)" kann man ein Bild einfügen

  • "src" ist ein Pflichtattribut und beinhaltet den Path bzw. die URL zum Bild
  • Alternativ falls das Bild nicht angezeigt werden kann: "alt", falls das Bild nicht Angezeigt werden kann, dann der "alt" Wert, als Text

Merke: in dem Moment, wo man eine Website aufruft, in der ein Bild ist, wie bei jedem Aufruf der website, das Bild über die "src" abgefragt,
daher vom selben Server oder einem anderen und fügt es einfach in die HTML-Seite ein. Falls also Bild nicht mehr über den eingetragenen "src" aufrufbar ist
bspw. wegen eines Broken Links, dann wird ein "Borken Link Icon" angezeigt.
