Constraints and Rules for Order:

TypeScript adds additional rules and constraints to JavaScript, particularly through static typing. These rules help detect errors early and improve the structure and maintainability of code.

Structural Type Checking:

In TypeScript, types are compared structurally, meaning that two types are considered equal if their structures are the same, regardless of their names. This facilitates working with different modules and libraries.

Void Return Type:

The return type void in TypeScript indicates that a function has no return values, and it is equivalent to undefined. This differs from some other statically typed languages where void is more specific.

Functions as Objects:

Functions are represented as objects in TypeScript, allowing functions to be passed as parameters and used as return values.

Method Overloading and Overriding:

TypeScript allows method overloading, where only one implementation can exist. Method overriding is possible without special annotations as in Java.

Optional Parameters and Non-Null Assertion:

TypeScript allows the use of optional parameters in functions (e.g., x?: number) and non-null assertion (x!.toString()) to inform the compiler that a variable will never be null or undefined.
